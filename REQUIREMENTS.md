# Poker Memo 要件定義書 v0.2

## 1. アプリ概要

Poker Memoは、カジノのポーカールームで同卓しているプレイヤーを素早く記録するPWA Webアプリである。

主な目的は、テーブル図から席をタップし、人物ごとの特徴、プレイスタイル、自由メモをアカウント単位で蓄積・再利用できるようにすること。

## 2. 対象環境

- iPhoneでの利用を主対象とする
- PWAとしてホーム画面追加に対応する
- 初期版はオンライン前提とする
- UIはiOS標準アプリに近い操作感を目指す
- デフォルトテーマはダークモード
- 設定画面でライト/ダークを切り替え可能

## 3. 技術方針

- フロントエンド: 静的HTML/CSS/JavaScript
- アプリ形式: PWA
- 認証: Supabase Auth
- データ保存: Supabase PostgreSQL
- アカウント単位でデータを分離する
- Row Level Securityでユーザーごとのアクセス制御を行う

## 4. 認証要件

### 対応する認証方式

- メールアドレス + パスワード
- Googleログイン
- Appleログイン

### アカウント機能

- サインアップ
- ログイン
- ログアウト
- パスワード再設定
- アカウント削除

## 5. 画面要件

### 5.1 ログイン画面

- メールアドレス + パスワードでログインできる
- 新規登録に切り替えられる
- Googleログインボタンを表示する
- Appleログインボタンを表示する
- パスワード再設定メールを送信できる

### 5.2 セッション一覧画面

- セッションを最近更新順で表示する
- 各セッションには以下を表示する
  - カジノ名
  - 席数
  - 着席人数
  - Hero席
  - 更新日時
- 新規セッションを作成できる
- セッションをタップするとテーブル画面へ遷移する

### 5.3 新規セッション作成画面

- カジノ名を入力する
- 席数を選択する
- デフォルト席数は9-max
- 対応席数は6-max / 8-max / 9-max / 10-max

### 5.4 テーブル画面

- ポーカーテーブルをレーストラック型に表示する
- テーブル上に席を配置する
- 各席をタップしてアクションシートを表示する
- 右上の一覧ボタンは表示しない
- Dealerに関する表示・機能は持たない

#### 空席をタップした場合

- 既存人物を選ぶ
- 新規人物を作る
- Hero席にする

#### 着席済みの席をタップした場合

- 人物詳細を見る
- 席から外す
- Hero席にする

#### テーブル上の席カード表示

- Seat番号
- Hero表示
- SVG人物アイコン
- ニックネームまたはUnknown
- 代表タイプタグ
- 代表行動タグ
- 国/地域マーカー
- プレイスタイルタグに応じたカード色

#### プレイスタイル別カード色

- TAG: 青
- LAG: 紫
- Nit: グレー
- Calling Station: アンバー
- Maniac: 赤
- Reg: 青緑
- Fish: 緑
- Unknown: ニュートラルグレー

### 5.5 人物検索/選択画面

- 既存人物を検索して席に追加できる
- 新規人物を作成できる
- 検索対象は以下
  - ニックネーム
  - 外見自由メモ
  - 自由メモ
  - プレイスタイルタグ
  - 過去に着席したカジノ名
- タイプタグで絞り込みできる
- 行動タグで絞り込みできる
- 検索結果には人物アイコン、名前、タグ、最終同卓情報を表示する

### 5.6 人物詳細/編集画面

- 1画面スクロール構成とする
- 右上に保存ボタンを表示する
- 画面下部にも保存ボタンを表示する
- 保存後は、セッション文脈がある場合はテーブル画面へ戻る
- 新規人物作成後は人物詳細/編集画面へ遷移する

編集項目:

- ニックネーム
- 国/地域マーカー
- 外見アイコン用パーツ
- 外見自由メモ
- 自由メモ
- 現在のタイプタグ
- 現在の行動タグ

削除操作:

- 人物を削除できる

### 5.7 設定画面

- ログイン中のユーザー情報を表示する
- ライト/ダークテーマを切り替えできる
- ログアウトできる
- パスワード再設定メールを送信できる
- アカウント削除ができる

## 6. 人物データ要件

人物はセッションをまたいで再利用できるプロフィールとして管理する。

### 人物プロフィール

- ニックネーム
- 国/地域マーカー
- SVG外見アイコン
- 外見自由メモ
- 自由メモ
- 現在のタイプタグ
- 現在の行動タグ
- 作成日時
- 更新日時

### 新規人物作成

- 必須入力はなし
- 空の人物を作成できる
- 初期表示名はUnknown
- 初期タイプタグはUnknown
- 空席から新規人物を作成した場合、作成後に人物詳細/編集画面へ遷移する

## 7. 外見アイコン要件

### 基本方針

- 写真は扱わない
- SVGアイコンのみを使う
- アイコンは選択式パーツの組み合わせで生成する
- 自由入力メモはSVG生成に影響しない
- プルダウン変更時にSVGプレビューを即時更新する

### 外見アイコン用パーツ

- 国/地域マーカー
- 髪型: 短髪 / 長髪 / 坊主 / なし
- 帽子: なし / キャップ / ニット帽
- 眼鏡: なし / 眼鏡 / サングラス
- ひげ: なし / 口ひげ / あごひげ / フル
- 服タイプ: Tシャツ / シャツ / パーカー / ジャケット
- 服色: 黒 / 白 / グレー / 赤 / 青 / 緑

### 削除済みの外見パーツ

- 肌トーン
- 髪色

### 国/地域マーカー

以下を選択できる。

- 日本人
- 中国人
- 韓国人
- その他アジア人
- 白人
- 黒人
- ラテン系

国/地域マーカーは、テーブル画面のカードに小さく表示する。SVGアイコンにはバッジとして反映する。

## 8. プレイスタイル要件

タイプタグと行動タグを組み合わせて管理する。

### タイプタグ

- TAG
- LAG
- Nit
- Calling Station
- Maniac
- Reg
- Fish
- Unknown

### 行動タグ

- VPIP多い
- VPIP少ない
- 3bet多い
- 3bet少ない
- C-bet多い
- ブラフ多い
- コール多い
- フォールド多い
- ショーダウン弱い
- リバー降りない

## 9. メモ要件

### 現行仕様

- 人物プロフィールに自由メモを持つ
- 人物詳細画面で自由メモを直接編集する
- メモ追記機能は持たない
- メモ履歴は持たない

### 削除済み仕様

- メモ追記アクション
- メモ追記フォーム
- メモ履歴表示
- `player_notes` テーブル

## 10. セッションデータ要件

### セッション

- カジノ名
- 席数
- Hero席
- 作成日時
- 更新日時

### セッション内着席情報

- セッションID
- Seat番号
- 人物ID

## 11. データベース要件

### テーブル

- `players`
- `sessions`
- `session_seats`

### 削除済みテーブル

- `player_notes`

### RLS

すべての主要テーブルでRLSを有効にし、`auth.uid() = user_id` のデータのみ操作できるようにする。

## 12. PWA/非機能要件

- PWA manifestを持つ
- Service Workerで静的アセットをキャッシュする
- キャッシュ更新時はバージョンを上げる
- 初期版はオンライン前提
- Supabase未設定時は設定案内を表示する

## 13. 初期版では扱わないもの

- 写真保存
- Dealer表示/Dealer位置設定
- LINEログイン
- オフライン編集
- データエクスポート
- データインポート
- セッションごとのゲーム種別
- ブラインド
- テーブル番号
- 収支
- メモ履歴
